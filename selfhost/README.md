# Portfolio98 — Self-Host on Linux Mint / Ubuntu / Debian

Everything lives in one Node.js process (API + frontend).  
PostgreSQL stores all data.

---

## Quick Start

```bash
# 1. Clone (or copy) this repo onto your server
git clone <your-repo-url> ~/portfolio98
cd ~/portfolio98

# 2. One-time setup (installs Node 24, pnpm, PostgreSQL, creates DB + .env)
bash selfhost/setup.sh

# 3. Start the site
bash selfhost/start.sh
# → open http://localhost:3000

# 4. (Optional) graphical launcher with Start/Stop/Log viewer
python3 selfhost/launcher.py

# 5. (Optional) add a desktop/app-menu shortcut for the launcher
bash selfhost/install-launcher-shortcut.sh
```

---

## Files

| File | Purpose |
|------|---------|
| `setup.sh` | One-time installer (Node, pnpm, PostgreSQL, DB, .env, build) |
| `start.sh` | Start everything in the background |
| `stop.sh` | Stop everything |
| `rebuild.sh` | Pull new code → rebuild → restart |
| `launcher.py` | Python/Tkinter GUI with Start/Stop/Restart/Log viewer |
| `install-launcher-shortcut.sh` | Add a desktop app icon for the launcher |
| `.env` | Your config (auto-generated by setup.sh, never committed) |
| `.env.example` | Template |

---

## Configuration (`selfhost/.env`)

```
DATABASE_URL=postgresql://portfolio98:password@localhost:5432/portfolio98
SESSION_SECRET=<random 64-char hex>
PORT=3000
SERVE_STATIC=1        # serve frontend + API from the same port
NODE_ENV=production
```

Run `setup.sh` once and it writes this file automatically with a generated password and secret.

## Updating

```bash
git pull
bash selfhost/rebuild.sh
```

Or click **Rebuild** in the launcher GUI.

## Expose to the internet (optional)

For HTTPS and a real domain, put nginx in front:

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

Then use Certbot for free HTTPS: `sudo certbot --nginx -d yourdomain.com`

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Port 3000 in use | Edit `PORT=` in `.env`, then restart |
| "API bundle not found" | Run `bash selfhost/rebuild.sh` |
| PostgreSQL won't start | `sudo systemctl status postgresql` |
| python3-tk missing | `sudo apt install python3-tk` |
| Launcher icon missing | Run `bash selfhost/install-launcher-shortcut.sh` |
